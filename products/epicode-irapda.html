One of the most sought after capabilities in an outbound dialer software is its ability to support the predictive dialer mode, especially for the Telemarketing and payment collections processes.

IraPDA is an API based predictive dialer algorithm and is a part of Epicode’s CTE ( Computer Telephony Enablers) range of middleware products. IraPDA is designed as a backend module that can be used by Systems Integrators or Services providers in the Customer engagement solution space.

Typically, Systems integrators in the Customer engagement space put together multiple applications like CRM/LMS, Telephony platform with inbound and outbound calling capabilities, WebRTC /Softphone with Agent UI and CTI , Workforce Management and analytics Software to provide an end to end solution for their customers.

IraPDA can be used by Systems integrators to add Predictive dialer capabilities to their Outbound module.

Integration Method
  • As part of the OEM integration, the Systems integrator partner can create a configuration page on their campaign manager web portal to manage the settings of IraPDA.
  • Access to API documentation will be provided for integration. APIs are available in various programming languages.
  • System Integrators can add the ability of calling these APIs at predefined intervals and sending parameters such as max_dialing_rate, tenant_id, aggr_factor, idle_low_water_mark, idle_high_water_mark etc from their campaign manager or LCM.
  • Based on the inputs received, IraPDA calculates the number of calls to be dialled in the next interval and sends the number back to the dialler.
  • Upon receipt of the response from IraPDA, the Campaign manager sets the dialling ratio. This helps in maintaining the call rate based on the availability of the agents.
